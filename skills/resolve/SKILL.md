# Resolve Skill

Use this skill when `execute` dispatches an annotation whose extraction file contains `# Request (resolve)`.

## Input

You will receive one extraction `.md` file generated by `v2/skills/execute/scripts/parse.py`.

Expected sections:
- `# Original File`
- `# Content to Modify`
- `# Parameters`
- `# Context` with optional `## <refname>` blocks
- One or more `# Request (...)` / `# Output` pairs (conversation history)
- `# Request (resolve)` (the final request)
- Optional: `# Hash`

## Interpretation of Fields

### Original File
- The source file for the annotation.
- Read this file to check whether surrounding edits have already addressed the annotation.

### Content to Modify
- **Full directive**: The original text between `@...@` sigils. This is the baseline content before any skill produced output.
- **Inline directive**: The surrounding paragraphs with a `@@` marker. Check whether the text around `@@` has already incorporated the proposed change.

### Parameters
- Ignore this section.

### Context
- Supplemental information. May help determine whether the annotation has been addressed by surrounding edits.

### Request / Output History
- The extraction file will typically contain earlier `# Request (skillname)` / `# Output` pairs before the final `# Request (resolve)`.
- The `# Output` from a prior skill is the proposed change to evaluate.
- Use this history to understand what was proposed and decide how to resolve it.

### Request (resolve)
- Typically empty or contains minimal instruction.
- The intent is: finalize the annotation by accepting or discarding the proposed output.

### Hash
- Ignore this section.

## Workflow

1. Read the extraction file once and locate all required sections.
2. Validate request:
   - If `# Request (resolve)` is missing, return:
   - `ERROR: missing # Request (resolve)`
3. Examine the conversation history:
   - Look for `# Output` sections from prior skill runs.
   - If a prior `# Output` exists, it contains the proposed change.
4. Decide resolution:
   - **If prior output exists**: Accept the proposed change. Return the output content as the resolved text, integrated with `# Content to Modify` as appropriate.
   - **If no prior output exists**: The annotation has no pending change. Return the original `# Content to Modify` unchanged.
   - **If surrounding context shows the change was already applied**: Return the original `# Content to Modify` unchanged.
5. Return the resolved content.
6. After `execute` writes the result back, `execute` will remove the annotation tags for this directive (and only this directive) from the input file.

## Output Contract

- Return the finalized content text.
- If accepting a proposed change, return the integrated result (the content as it should appear in the document).
- If no change is needed, return the original `# Content to Modify` text unchanged.
- Do not wrap output in `<output ...>` tags.
- Do not modify files directly; `execute` performs all write-back actions and annotation removal.
