# Cite Skill

Use this skill when `execute` dispatches an annotation whose extraction file contains `# Request (cite)`.

## Input

You will receive one extraction `.md` file generated by `v2/skills/execute/scripts/parse.py`.

Expected sections:
- `# Original File`
- `# Content to Modify`
- `# Parameters`
- `# Context` with optional `## <refname>` blocks
- `# Request (cite)`
- Optional: `# Output`, `# Hash`

## Interpretation of Fields

### Original File
- The source file for the annotation.
- Read this file to determine the document format (LaTeX, Markdown, etc.) and to find existing `.bib` files or reference sections in the project.

### Content to Modify
- **Full directive**: The text between `@...@` sigils contains the claim that needs a citation. Do not modify this content; find a citation for it.
- **Inline directive**: The content includes surrounding paragraphs with a `@@` marker. The claim needing citation is the text near the `@@` marker. Use surrounding paragraphs to understand the claim in context.

### Parameters
- Check for `source` parameter indicating where to search for citations:
  - `source:arxiv` — search arXiv
  - `source:gscholar` — search Google Scholar
  - `source:web` — general web search
  - `source:local` — search local `.bib` files or reference sections in the project
  - If not specified, use your best judgment based on the content (academic claims default to scholarly sources).

### Context
- Supplemental information that may help identify the correct source.
- May include related references or bibliography entries.

### Request (cite)
- The instruction to satisfy. It may specify citation style (e.g., `<cite APA>`, `<cite IEEE>`, `<cite bibtex>`).
- If the instruction is empty, infer the appropriate format from the document type.

### Output
- If `# Output` is the final section, treat it as stale prior output and regenerate.
- If another `# Request (...)` appears after `# Output`, treat prior output as conversation history.

### Hash
- Ignore this section.

## Workflow

1. Read the extraction file once and locate all required sections.
2. Validate request:
   - If `# Request (cite)` is missing, return:
   - `ERROR: missing # Request (cite)`
3. Determine document format:
   - Read `# Original File` to check if the project is LaTeX (`.tex`), Markdown, or another format.
   - If LaTeX, look for existing `.bib` files in the project directory.
4. Identify the claim needing citation from `# Content to Modify`.
5. Search for the original source:
   - Check `# Parameters` for preferred search source.
   - Search scholarly databases, web, or local `.bib` files as appropriate.
   - Prioritize primary sources over secondary references.
6. Format the citation:
   - If the user specified a style in `# Request (cite)`, use that style.
   - If the project is LaTeX and a `.bib` file exists:
     - Check if the source already exists in the `.bib` file. If so, return `\cite{existing_key}`.
     - If not, return the `\cite{key}` command and provide the BibTeX entry to add.
   - If no style is specified, infer from the document format (LaTeX → `\cite{}`, Markdown → inline link or footnote, etc.).
7. Return the formatted citation. Tell the main agent the default response mode is replace (directly add the citation to the statement, if it is latex project, should also add citation to `.bib` and use `\cite{}` in the content). Also notify the main agent to remove the if this is an inline directive.

## Output Contract

- Return the citation string ready to be inserted into the document.
- For LaTeX projects with existing `.bib` entries, return just `\cite{key}`.
- For LaTeX projects needing a new entry, return both the `\cite{key}` and the BibTeX entry:
  ```
  \cite{author2024title}

  BibTeX entry:
  @article{author2024title,
    author = {...},
    title = {...},
    ...
  }
  ```
- For non-LaTeX projects, return the formatted citation string in the requested or inferred style.
- Do not wrap output in `<output ...>` tags.
- Do not modify files directly; `execute` performs all write-back actions.
