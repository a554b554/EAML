# Placeholder Skill

Use this skill when `execute` dispatches an annotation whose extraction file contains `# Request (placeholder)` or `# Request (ph)`.

## Input

You will receive one extraction `.md` file generated by `v2/skills/execute/scripts/parse.py`.

Expected sections:
- `# Original File`
- `# Content to Modify`
- `# Parameters`
- `# Context` with optional `## <refname>` blocks
- `# Request (placeholder)` or `# Request (ph)`
- Optional: `# Output`, `# Hash`

## Interpretation of Fields

### Original File
- The source file for the annotation.
- Read this file if you need broader document context for tone or content cues.

### Content to Modify
- **Inline directive only.** The content includes the paragraph before, the current paragraph, and the paragraph after, with a `@@` marker at the exact position where the placeholder should be filled in.
- Do not modify the surrounding text. Generate only the content to insert at the `@@` position.
- If this is a full directive (no `@@` marker, content between `@...@` sigils), return:
  `ERROR: placeholder skill only supports inline directives`

### Parameters
- Ignore this section unless the user request explicitly depends on a parameter value.

### Context
- Supplemental information for the request.
- Use context blocks to match tone, style, and content of the surrounding document.

### Request (placeholder / ph)
- The instruction describing what to generate (e.g., `a one-sentence summary of the intro`, `a strong ending`, `a fancy word`).
- Treat it as the highest-priority directive.

### Output
- If `# Output` is the final section, treat it as stale prior output and regenerate.
- If another `# Request (...)` appears after `# Output`, treat prior output as conversation history.

### Hash
- Ignore this section.

## Workflow

1. Read the extraction file once and locate all required sections.
2. Validate:
   - If `# Request (placeholder)` or `# Request (ph)` is missing, return:
   - `ERROR: missing # Request (placeholder)`
   - If `# Content to Modify` does not contain a `@@` marker, return:
   - `ERROR: placeholder skill only supports inline directives`
3. Read the surrounding text in `# Content to Modify`:
   - The paragraph before `@@` provides leading context.
   - The paragraph after `@@` provides trailing context.
   - Use `# Context` blocks for additional tone and content cues.
4. Generate content that:
   - Satisfies the user's instruction in `# Request`
   - Fits naturally at the `@@` position in terms of tone, style, and flow
   - Matches the document's language and format
   - Is concise and self-contained — it should read as if it was always part of the text
5. Return only the generated content (not the surrounding paragraphs).

## Output Contract

- Return only the generated placeholder text.
- Do not include the surrounding paragraphs — just the content to insert at `@@`.
- Do not wrap output in `<output ...>` tags.
- Do not modify files directly; `execute` performs all write-back actions.
