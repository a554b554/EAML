# Prompt Skill

Use this skill when `execute` dispatches an annotation whose extraction file contains `# Request (prompt)`.

## Input

You will receive one extraction `.md` file generated by `v2/skills/execute/scripts/parse.py`.

Expected sections:
- `# Original File`
- `# Content to Modify`
- `# Parameters`
- `# Context` with optional `## <refname>` blocks
- `# Request (prompt)`
- Optional: `# Output`, `# Hash`

## Interpretation of Fields

### Original File
- This is the source file for the annotation.
- If the extraction file lacks needed information, you may read this file for additional context.
- You could also further refer to additional files that described in the original file if necessary.

### Content to Modify
- **Full directive**: Only this content is editable. If protected segments like `<<...>>` are present, preserve the protected text exactly.
- **Inline directive**: The content includes the paragraph before, the current paragraph, and the paragraph after. A `@@` marker indicates the exact position where the user placed the inline request, but you are not required to place the output at that exact location. Use the surrounding paragraphs for context for resolve the request; You can either insert content at any position or modify text in the paragraphs as needed to satisfy the request.

### Parameters
- Ignore this section unless the user request explicitly depends on a parameter value.

### Context
- Supplemental information for the request.
- Use context blocks when relevant; they may include copied text, local-file content, or fetched references.

### Request (prompt)
- This is the instruction to satisfy.
- Treat it as the highest-priority directive.

### Output
- If `# Output` is the final section, treat it as stale prior output and ignore it.
- If another `# Request (...)` appears after `# Output`, treat prior output as conversation history.

### Hash
- Ignore this section.

## Workflow

1. Read the extraction file once and locate all required sections.
2. Validate request:
   - If `# Request (prompt)` is missing, return:
   - `ERROR: missing # Request (prompt)`
3. Build working inputs:
   - `instruction` from `# Request (prompt)`
   - `source_text` from `# Content to Modify` (or inline placeholder mode)
   - `supporting_context` from `# Context` subsection content
4. Produce response text that:
   - Follows the request exactly
   - Preserves protected text semantics
   - Uses provided context when helpful
   - Avoids fabricating unsupported facts
5. Return only final response text to the caller (`execute`).

## Output Contract

- Return plain text only.
- Do not wrap output in tags like `<output ...>`.
- Do not add markdown fences unless the request explicitly asks for code blocks.
- Do not modify files directly; `execute` performs all write-back actions.
